שלב 1

יצירת Scenario ב-Make.com

1
נכנסו ל-make.com וצרו Scenario חדש.
2
הוסיפו Module ראשון — Trigger שמפעיל את הpreזרימה (לדוגמה: Google Sheets, Webhook, or Schedule).
3
הוסיפו Module שני — חפשו "HTTP" ← בחרו "Make a Request".
🔧
שלב 2

הגדרת HTTP Module לשליחת הודעה

1
URL: הכניסו:
https://graph.facebook.com/v18.0/{PHONE_NUMBER_ID}/messages
(החליפו {PHONE_NUMBER_ID} במספר שלכם)
2
Method: בחרו POST
3
Headers: הוסיפו Header:
Name: Authorization
Value: Bearer {ACCESS_TOKEN}
4
Body Type: בחרו Raw / JSON ← הכניסו את ה-JSON מהדוגמה מטה.
JSON Body — Make HTTP Module 
{
  "messaging_product": "whatsapp",
  "to": "{{1.phone}}",        // מהmodule הקודם
  "type": "text",
  "text": {
    "body": "שלום {{1.name}}, ההזמנה שלך אושרה! 🎉"
  }
}
💡
הערכים {{1.phone}} ו-{{1.name}} הם משתנים שמגיעים מה-Module הקודם בScenario. אפשר לשנות לכל שדה שרלוונטי לכם.
▶️
שלב 3

הרצת בדיקה

1
לחצו Run once ← ה-Scenario ירוץ פעם אחת.
2
אם הצליח — תראו סימון ירוק על ה-HTTP Module ותקבלו הודעת WhatsApp.
3
אם נכשל — לחצו על ה-Module ← Output לראות את שגיאת Meta (לרוב token שגוי או phone_number_id לא נכון).
🔧
n8n: הלוגיקה זהה ל-Make — שימוש ב-HTTP Request Node עם אותם פרטים.
🔧
שלב 1

הוספת HTTP Request Node

1
ב-n8n Workflow, הוסיפו Node ← חפשו "HTTP Request".
2
Method: POST | URL: https://graph.facebook.com/v18.0/PHONE_ID/messages
3
ב-Authentication: בחרו "Header Auth", הוסיפו Header: Authorization: Bearer TOKEN
4
Body: JSON — אותו body כמו ב-Make.
🔑
טיפ n8n

שמירת Credentials בצורה בטוחה

🔐
ב-n8n, מומלץ לשמור את ה-Access Token כ-Credential (לא ישירות ב-Node) — ב-Credentials Manager ← New Credential ← Header Auth. כך הtoken מוצפן ולא נחשף בexport.
🌐
שליחת הודעת טקסט

HTTP POST ישיר ל-Meta API

שליחת הודעת טקסט
POST https://graph.facebook.com/v18.0/{PHONE_NUMBER_ID}/messages
Authorization: Bearer {ACCESS_TOKEN}
Content-Type: application/json

{
  "messaging_product": "whatsapp",
  "to": "972501234567",
  "type": "text",
  "text": { "body": "שלום מAutoScale! 👋" }
}
שליחת הודעת Template
{
  "messaging_product": "whatsapp",
  "to": "972501234567",
  "type": "template",
  "template": {
    "name": "hello_world",    // שם הtemplate שאישרה Meta
    "language": { "code": "he" },  // he לעברית, en_US לאנגלית
    "components": [
      {
        "type": "body",
        "parameters": [
          { "type": "text", "text": "שם הלקוח" }
        ]
      }
    ]
  }
}
תגובה מוצלחת מ-Meta (200 OK)
{
  "messaging_product": "whatsapp",
  "contacts": [{ "input": "972501234567", "wa_id": "972501234567" }],
  "messages": [{ "id": "wamid.HBgM..." }]
}
📡
Webhook נכנס — מאפשר לכם לקבל הודעות שלקוחות שולחים. Meta שולחת POST לכתובת שלכם בכל הודעה נכנסת.
🔗
שלב 1

הגדרת Webhook URL ב-Meta

1
ב-Meta Developers ← האפליקציה שלכם ← WhatsApp ← Configuration.
2
ב-Webhook, לחצו Edit.
3
Callback URL: הכניסו את ה-URL של ה-webhook שלכם (מ-Make, n8n, או השרת שלכם).
4
Verify Token: הכניסו מחרוזת סודית שתבחרו (לדוגמה: mySecretToken123). שמרו אותה — תצטרכו אותה בשלב הבא.
5
לחצו Verify and Save — Meta שולחת GET עם הtoken לאימות. ה-Webhook שלכם חייב להחזיר אותו.
⚠️
ב-Make.com: צרו Custom Webhook Module כטריגר — Make נותן לכם URL ייחודי. שימוש ב-URL הזה כ-Callback URL ב-Meta.
📥
שלב 2

מבנה ההודעה הנכנסת מ-Meta

Webhook Payload — הודעת טקסט נכנסת
{
  "object": "whatsapp_business_account",
  "entry": [{
    "changes": [{
      "value": {
        "messages": [{
          "from": "972501234567",    // מספר השולח
          "type": "text",
          "text": { "body": "הטקסט שהלקוח שלח" },
          "timestamp": "1234567890"
        }],
        "contacts": [{
          "profile": { "name": "שם הלקוח" },
          "wa_id": "972501234567"
        }]
      }
    }]
  }]
}
✅
ב-Make: אחרי קבלת ה-Webhook — השתמשו ב-{{1.entry[].changes[].value.messages[].from}} לקבלת מספר השולח, ו-{{1.entry[].changes[].value.messages[].text.body}} לקבלת תוכן ההודעה.
